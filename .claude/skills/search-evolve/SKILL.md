# 搜索策略每日进化

## 核心原则
- **自动进化，不需用户确认** — 自己调整策略，用户看报告即可
- **用户反馈是最重要的信号** — 如果用户发了一篇我没搜到的报道，必须分析原因并调整策略
- 目标：让每日搜索越来越强，最终找到 100 篇报道

## 操作流程

### 1. 收集数据

读取：
- `search-log.jsonl` — 今日及历史搜索日志
- `sources/TopicReportList.MD` — 所有已收录报道
- `search-strategy.md` — 当前策略（如果存在）
- `goals.md` — 目标和进度
- `.claude/agents/media-monitor.md` — agent 知识库

### 2. 分析搜索效果

**关键词效果**：
- 哪些关键词命中率高（保留、加强）
- 哪些关键词总是返回已有结果（降权或换时间段）
- 是否需要新增关键词变体

**来源分析**：
- 已收录报道的来源分布（哪些网站反复出现）
- 今日搜索中发现的新来源
- 哪些高产来源还没做定向搜索

**搜索角度分析**：
- 今天用的搜索角度效果如何
- 明天应该用什么角度

### 3. 消化用户反馈

检查是否有用户反馈的"漏搜"报道（可能记录在 search-log.jsonl 的 notes 中或单独的反馈文件）。

如果有，重点分析：
- **为什么没搜到？**
  - 关键词没覆盖？→ 补关键词
  - 来源没覆盖？→ 加来源
  - 搜索引擎没收录？→ 换搜索渠道
  - 文章措辞不同？（没提名字只提职位）→ 加变体关键词
- **怎么确保下次能搜到类似的？**

### 4. 联网学习

搜索以下内容发现新机会：
- 新能源/充电行业近期有什么大会、论坛
- 行业媒体有无新的平台或频道
- 搜索技巧：如何更有效地搜索中文人名相关报道

### 5. 更新策略文件

将所有分析结果写入 `search-strategy.md`：

```markdown

## 上次更新
[日期]

## 关键词
### 核心（每次必搜）
- ...
### 变体（轮换使用）
- ...
### 从用户反馈中学到的
- ...

## 高产来源
| 来源 | 命中次数 | URL 模式 | 最后命中 |
|------|---------|----------|---------|

## 搜索角度轮换计划
- 周一：...
- 周二：...
- ...

## 近期关注
- 即将举办的行业大会
- 近期热点话题

## 经验教训
### 有效的搜索方法
- ...
### 失败的搜索方法（避免重复）
- ...
### 从用户反馈中学到的
- [日期] 用户反馈漏搜 [URL]，原因：...，已调整：...

## 改进历史
| 日期 | 改进 | 原因 |
|------|------|------|
```

### 6. 更新 Agent 知识库

如果有重要发现，同步更新 `.claude/agents/media-monitor.md` 中的：
- 高产来源表
- 关键词列表
- 搜索技巧

### 7. 更新目标进度

读取 `goals.md`，更新进度追踪表格。

### 8. 输出报告

```

📊 今日搜索效果
- 搜索 X 次，返回 X 条，新发现 X 篇
- 最有效关键词：...
- 最有效来源：...

📝 今日学到
- [具体发现/教训]

✏️ 策略调整
- [做了什么改变]

🎯 进度：XX / 100（XX%）
```
